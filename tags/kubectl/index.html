Managing different kubectl and helm versions for different clusters can be a pain. Seems asdf is pretty cool for managing the versions and a simple shell alias can do all the setup. alias k8s_local='asdf local kubectl 1.20.0 && \ asdf local helm 3.4.2 && \ asdf current && \ kctx docker-desktop && \ kubeon' alias k8s_ci='asdf local kubectl 1.19.4 && \ asdf local helm 2.12.3 && \ asdf current && \ kctx ci && \ kubeon' $ k8s_local $ k8s_ci
